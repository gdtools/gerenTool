# 滚动截图功能文档

## 1 功能概述

滚动截图是一种高级截图功能，允许用户捕获超出屏幕可视区域的完整网页或长页面内容。该功能通过模拟人工滚动操作，逐帧捕获页面内容并拼接成一张完整的长图。

---

## 2 入口点

### 2.1 入口触发位置

滚动截图的入口位于 `src/screenshot/feature/screenshot/mod.rs` 中的 `ScreenshotFeature` 结构体。当用户执行截图操作并选择"滚动截图"时，代码通过 `ScreenshotAction::ScrollCapture` 枚举值触发滚动截图流程。

```rust
/// src/screenshot/feature/screenshot/mod.rs
/// 滚动截图入口点
match action {
    ScreenshotAction::ScrollCapture => {
        // 获取用户选择的选区矩形
        if let Some(selection) = self.state.select.selection {
            // 创建控制通道（用于发送停止命令）
            let (control_tx, control_rx) = std::sync::mpsc::channel();
            // 创建结果通道（用于接收截图结果）
            let (result_tx, result_rx) = std::sync::mpsc::channel();
            // 保存通道引用到状态中
            self.scroll_control_tx = Some(control_tx);
            self.scroll_result_rx = Some(result_rx);
            // 启动滚动截图线程，传入窗口句柄、选区、控制通道和结果通道
            scroll_capture::start_scroll_capture_thread(
                common.window_state.hwnd_usize,
                selection,
                control_rx,
                result_tx,
            );
        }
    }
    // 停止滚动截图
    ScreenshotAction::StopScrollCapture => {
        if let Some(tx) = &self.scroll_control_tx {
            let _ = tx.send(scroll_capture::ScrollControlMessage::Stop);
        }
    }
}
```

## 3 实现原理

### 3.1 核心实现原理

滚动截图的实现原理可以分为以下几个关键步骤：

#### 步骤一：覆盖层全屏透明化选区

在用户选择截图区域后，保持截图覆盖层全屏显示，但将选区内部区域透明化。这一步的目的是让用户能够看到下方窗口的内容，同时接收滚轮事件。

#### 步骤二：Z-Order 遍历穿透滚轮事件

使用 Windows 的 Z-Order（窗口层次顺序）遍历，找到选区下方对应的窗口。然后将滚轮事件（WM_MOUSEWHEEL）穿透发送给该窗口，模拟用户手动滚动的行为。

关键实现代码：
```rust
/// src/screenshot/feature/screenshot/scroll_capture.rs
/// 滚动截图实现原理：
/// 1. 保持截图覆盖层全屏，但选区内部透明化，使用 Z-Order 遍历将滚轮事件穿透发送给下方窗口
/// 2. 模拟鼠标滚轮向下滚动，等待页面响应
/// 3. 用选区裁剪屏幕，通过比较相邻帧的重叠区域计算滚动偏移量
/// 4. 将新内容拼接到长图底部
/// 5. 重复直到检测到页面不再滚动、鼠标移出选区或点击停止按钮
/// 6. 合并后保存最终长图到桌面
```

#### 步骤三：帧间滚动偏移量计算

每次滚动后，使用模板匹配算法比较相邻两帧的重叠区域，计算出精确的滚动偏移量。算法流程如下：

1. 截取当前帧的选区图像
2. 与上一帧的下半部分进行像素对比
3. 找到完全匹配的区域（重叠区域）
4. 根据重叠区域的位置计算出滚动偏移量

相关常量定义：
```rust
/// 每次滚动的滚轮格数（负数表示向下滚动）
pub const SCROLL_DELTA: i32 = -4;
/// 每次滚动后等待页面响应的时间（毫秒）
pub const SCROLL_WAIT_MS: u64 = 400;
/// 用于检测重叠区域的比较行数
const OVERLAP_SEARCH_ROWS: u32 = 80;
/// 像素差异阈值（0-255，超过此值认为像素不同）
const PIXEL_DIFF_THRESHOLD: u8 = 15;
/// 判断页面停止滚动的最小偏移量（像素）
pub const MIN_SCROLL_OFFSET: u32 = 5;
```

#### 步骤四：图像拼接

将新捕获的帧（去除与上一帧的重叠部分）拼接到长图底部。重复步骤二到四，直到检测到以下任一终止条件：

- 页面不再滚动（滚动偏移量小于 MIN_SCROLL_OFFSET）
- 鼠标移出选区
- 用户点击停止按钮

#### 步骤五：保存最终长图

将所有帧合并后的长图保存到用户桌面，完成整个滚动截图流程。

### 3.2 线程间通信

滚动截图在一个独立的线程中执行，通过 `std::sync::mpsc` 通道进行线程间通信：

```rust
/// 控制消息枚举
pub enum ScrollControlMessage {

/// 结果消息枚举
```

## 6 关键技术细节

### 6.1 窗口消息投递

使用 `SendMessage` Win32 API 将滚轮事件发送到目标窗口：
```rust
/// 发送 WM_MOUSEWHEEL 消息到目标窗口
/// wParam: 滚轮增量 (HIWORD) 和按键状态 (LOWORD)
/// lParam: 鼠标位置
SendMessageW(hwnd, WM_MOUSEWHEEL, wparam, lparam);
```

### 6.2 像素比较算法

滚动偏移量的计算基于像素级别的比较：
- 比较相邻两帧的重叠区域
- 使用 `OVERLAP_SEARCH_ROWS` 定义比较的行数
- 使用 `PIXEL_DIFF_THRESHOLD` 容忍轻微的像素差异
- 找到完全匹配的位置后，计算偏移量

### 6.3 图像拼接

使用 `image` crate 的 `ImageBuffer` 进行图像拼接：
- 创建足够大的目标缓冲区存储完整长图
- 逐帧将新内容复制到目标缓冲区的底部
- 处理图像边缘和对齐问题

---

## 7 总结

滚动截图功能通过以下技术手段实现长页面捕获：

1. **事件穿透**：利用 Windows Z-Order 和消息投递机制，模拟用户滚轮操作
2. **帧间计算**：通过像素比较算法精确计算每次滚动的偏移量
3. **图像拼接**：使用 image 库将多帧内容无缝拼接成完整长图
4. **线程通信**：使用 mpsc 通道实现主线程与截图线程的安全通信

该实现不依赖第三方滚动截图库，而是基于 Windows API 从零实现，提供了更好的控制性和定制能力。
